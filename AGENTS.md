# Repository Guidelines

## Project Structure & Module Organization
`Scrapile.slnx` ties together a layered .NET desktop app:
- `Scrapile.Domain/`: core entities and interfaces only.
- `Scrapile.Application/`: business services, DTOs, helpers.
- `Scrapile.Infrastructure/`: file-system and JSON persistence implementations.
- `Scrapile.Desktop/`: Avalonia UI (`Views/`, `ViewModels/`, DI wiring).
- `Scrapile.*.Tests/`: xUnit test projects aligned to their runtime layer.
- `docs/`: planning and archive notes.
- `pub/`: publish output generated by scripts (do not hand-edit).

Keep dependencies flowing inward (Desktop -> Application -> Domain; Infrastructure implements Domain interfaces).

## Versioning

Version is defined in `Scrapile.Desktop/Scrapile.Desktop.csproj`. When updating, change all three properties together:
- `<Version>` - Assembly version
- `<CFBundleVersion>` - macOS bundle version
- `<CFBundleShortVersionString>` - macOS display version

Version scheme: **major.minor.patch**
- **Minor** (second place): new or changed features
- **Patch** (third place): bug fixes

## Build, Test, and Development Commands
- `./publish.sh`: produce macOS/Windows release artifacts in `pub/` (self-contained and slim variants).

## Testing Guidelines
Tests use xUnit with `coverlet.collector` installed in test projects.
- Follow Arrange/Act/Assert sections for readability.
- Add/adjust tests with any behavior change in services, repositories, or view-model interactions.
